# RAG Service

`rag-service` is the internal FastAPI retrieval service used by [Smart Kiosk Assistant](../README.md). It ingests store or restaurant knowledge, embeds and persists that content in Chroma, retrieves the most relevant chunks for a user question, and streams grounded answers generated by an OpenVINO-backed LLM.

This service listens on port `8020` and can run either as part of the five-service kiosk stack or on its own for RAG-focused development and testing.

## Responsibilities

- Accept knowledge-base ingestion as raw text or uploaded `.txt` and `.md` files.
- Persist embeddings and metadata in a local Chroma collection.
- Retrieve relevant chunks for each query, apply runtime context, and construct the answer prompt.
- Stream answers to kiosk-core over Server-Sent Events.
- Expose an OpenAI-compatible chat endpoint for tool and client compatibility.

## Where It Fits In The Stack

Within the top-level compose deployment, `kiosk-core` treats `rag-service` as its answer-generation backend:

1. `kiosk-core` receives a transcription from the browser-driven voice session.
2. `kiosk-core` sends the question and optional runtime context to `POST /api/v1/query`.
3. `rag-service` retrieves from Chroma, prompts the LLM, and streams tokens back.
4. `kiosk-core` forwards the answer text to text-to-speech and returns the completed interaction to the UI.

## API Surface

### Core Endpoints

| Endpoint | Purpose |
|---|---|
| `GET /health` | Liveness probe for compose health checks and local verification |
| `POST /api/v1/context` | Ingest raw text into the active collection |
| `POST /api/v1/context/file` | Ingest a `.txt` or `.md` upload |
| `GET /api/v1/context/stats` | Inspect collection metadata and document counts |
| `DELETE /api/v1/context` | Clear the active collection |
| `POST /api/v1/query` | Stream a grounded answer for kiosk-core |
| `POST /v1/chat/completions` | OpenAI-compatible chat interface |

### Query Behavior

- `POST /api/v1/query` returns `text/event-stream`.
- Token fragments are emitted as SSE `data:` messages.
- A final optional `sources` event can be emitted when `include_sources=true`.
- The stream terminates with `data: [DONE]`.

## Current Default Runtime

The checked-in [config.yaml](config.yaml) currently defaults to:

- LLM: `Qwen/Qwen3-4B-Instruct-2507`
- LLM device: `GPU`
- Embeddings: `BAAI/bge-large-en-v1.5`
- Retrieval: `top_k=3`, `fetch_k=6`
- Context budget: `max_context_chars=8000`
- Storage collection: `smart-kiosk-assistant-bge-large`

These defaults are intentionally conservative for a kiosk flow: short retrieved context, concise answer generation, and persistent local storage.

## Storage And Runtime State

- LLM assets are cached under [models/](models/).
- Embedding assets are cached under [models/](models/).
- Vector data persists under [storage/](storage/), with Chroma stored in `storage/vector_db/`.
- OpenVINO cache data is written under `storage/ov_cache` when enabled by config.
- Chunk debug output can be written under `storage/chunks_debug/` when configured.
